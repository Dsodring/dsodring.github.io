Building My Portolio

7th October, 2019

Daniel Sødring

Here I go into more detail about how I have built my portfolio, what technologies and methods I have used have used and any design decisions I have made.

Design

Before designing my portfolio I browsed through Dribbble for inspiration and to see what is new and trending in web design. A thing I took note of was the use of bright colours and vivid drawings in website design. I decided to go with a slightly toned down colour palette. Using pastel colours throughout my website to achieve a warmer and friendlier look, while still maintaining a professional appearance with the fonts Montserrat for headers and Open Sans for body text.

I found an article, talking about a ‘broken’ grid layout with CSS Grid. I Tried to replicate a similar look in my mobile and desktop layout of my portfolio.

For my landing page I wanted to keep it simple and without a lot of text. The goal was for visitors to easily identify and understand who I am and what I do.

Designing my blog page I wanted to use larger pictures to catch the eye follow by the title and a short description of the blog posts content. I initially had the idea of a side navigation for all my blog posts but when coding my website I realised the three blog posts did not take up a lot of space so I decided not to use the menu and only implement it if more blog posts are added in the future.

For my project page I wanted to keep it even more simple and clean by only using a large image with the project title, hoping the image or graphic will intrigue the user to investigate and enter the subpage to read more about the project.

I kept the contact page light and simple with a HTML contact form from Formspree.io, coded by Derren Wilson and changed the style to fit the overall look of my website. Initially my HTML consisted of the input name and a bottom border, but in a study done by Google’s Material Design on text fields, they concluded text fields should either include a semi-transparent fill with a bottom border or a stroke and rounded corners, among other key findings (Zarasky et al., 2019), so I later changed it to conform to their findings.

Build

I started building my website in VS Code with a mobile first approach without the use CSS flexbox or grid, as without it, the page is already fully responsive and fits a mobile screen really well with minimal styling. I initially had my navigation at the top like on desktop websites, but in a talk on Designing for Speed & Hacking User Perception, Mustafa Kurtuldu talks about different styles of navigation including hamburger menu, backdrop drawer and top taps. In his talk he outlines how Facebook successfully implemented a bottom navigation and saw an increase in perception of speed, satisfaction and engagement (Conversions, 2019), and it obviously makes sense to put the navigation on the bottom on smaller devices such as smartphones as we can only move our thumbs so far up the screen before it gets uncomfortable.

I used flexbox and grid to design my layout on screens with a width larger than 900px by using media queries and by using a nested grid layout I could replicate the ‘broken grid layout’. I decided to only have the H1 and links break the grid as it would otherwise seem too cluttered and the smaller text would be difficult to read as it would be overlapping with images.

Before deciding on a final colour scheme I ran tests with the WebAIM Contrast Checker, I initially passed the AA check on small text with the original pastel red background and lighter grey paragraph, but after changing the paragraph colour to #222 it passed with the WCAG AAA.

I used Swup which is a JavaScript library to animate my page transitions. I decided to use page transitions because they are not only aesthetically pleasing but they also provide visual continuity and positive feedback (De Rosa, 2016). One of my concerns when including the Swup.JS library was speed, but in a test done on Firefox, the two script files were only 5.35KB and 419 B in size so it did not have any noticeable impact on the loading times.

Launch

I have all my files and images saved in my GitHub repository to make sure they are kept safe and I can edit them both at home and university. For hosting my website I have it on danielsodring.com and 34SP.com under the domain http://17085190.webdevmmu.uk/portfolio/
